# Ignite 开发指南

> 4 步学会如何在模板上添加新功能。即使你是新手，也能看懂。

---

## 📋 你要做什么？

在模板上添加一个新功能（比如"记事本"），只需要 **4 个步骤**：

```
步骤 1 → 定义数据库（存什么数据）
步骤 2 → 写后端 API（如何操作数据）
步骤 3 → 写 Hook（把操作包装成函数）
步骤 4 → 写页面（用户看到的界面）
步骤 5 → 添加到导航栏（让用户能找到）
```

## 快速上手：Next.js 是怎么工作的？

**Next.js 是什么？**

Next.js 是一个 React 框架，让你用 React 写网页，但它帮你处理了很多复杂的事情。

**核心概念（5 分钟看懂）：**

### 1. 文件即路由

```
文件位置                   →  访问 URL
app/page.tsx              →  http://localhost:3000
app/dashboard/page.tsx     →  http://localhost:3000/dashboard
app/dashboard/notes/page.tsx →  http://localhost:3000/dashboard/notes
```

**规则**：`app` 文件夹下的文件会自动变成网页地址。

### 2. 服务端组件 vs 客户端组件

```typescript
// 默认：服务端组件（可以省略 'use client'）
// 优势：SEO 好、加载快
export default function Page() {
  return <div>内容</div>
}
```

```typescript
// 客户端组件（必须写 'use client'）
// 何时用：需要使用 useState、useEffect、事件处理
'use client'
export default function Page() {
  const [count, setCount] = useState(0)
  return <div onClick={() => setCount(count + 1)}>{count}</div>
}
```

**简单判断**：如果你要写 `useState`、`onClick` 等，就加 `'use client'`。

### 3. layout.tsx - 共享布局

```
app/
├── layout.tsx        → 所有页面的外层布局
├── page.tsx          → 首页
└── dashboard/
    ├── layout.tsx    → dashboard 下的外层布局
    └── page.tsx      → dashboard 首页
```

**作用**：layout.tsx 里的内容会包裹它下面所有页面。

**本项目中的布局**：
- `app/layout.tsx` - 全局布局（Provider、主题等）
- `app/dashboard/layout.tsx` - 后台布局（左侧导航栏 + 顶部栏）

### 4. 左侧导航栏在哪？

**文件位置**：`app/dashboard/layout.tsx`

打开这个文件，你会看到导航项的定义，添加新功能时需要在这里添加链接。

## 步骤 1：定义数据库

**问题**：笔记需要存储什么信息？

**答案**：
- 标题（title）
- 内容（content）
- 谁写的（userId）
- 什么时候写的（createdAt）

**操作**：

1. 打开 `prisma/schema.prisma`
2. 在最下面添加：

```prisma
model note {
  id        String   @id @default(uuid())
  title     String
  content   String?
  userId    String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user user @relation(fields: [userId], references: [id], onDelete: Cascade)
}
```

3. 在 `user` 模型里添加 `notes note[]`

4. 保存后运行：
```bash
pnpm prisma migrate dev --name add_notes
```

**完成！** 数据库表已创建。

## 步骤 2：写后端 API

**问题**：如何操作数据库？

**答案**：写 API 接口，像服务员一样接收请求、操作数据。

**操作**：

1. 创建文件 `server/api/routes/notes/index.ts`：

```typescript
import { Hono } from 'hono'
import { z } from 'zod'
import { zValidator } from '@/server/api/validator'
import { prisma } from '@/lib/db'
import { auth } from '@/lib/auth'

const app = new Hono().basePath('/notes')

// 获取当前用户 ID
async function getUserId(c: any) {
  const session = await auth.api.getSession({ headers: c.req.raw.headers })
  return session?.user?.id || null
}

// 获取笔记列表
app.get('/', async (c) => {
  const userId = await getUserId(c)
  const notes = await prisma.note.findMany({
    where: { userId },
    orderBy: { createdAt: 'desc' }
  })
  return c.json({ data: notes })
})

// 创建笔记
app.post('/', zValidator('json', z.object({
  title: z.string().min(1),
  content: z.string().optional(),
})), async (c) => {
  const userId = await getUserId(c)
  const { title, content } = c.req.valid('json')
  const note = await prisma.note.create({
    data: { title, content, userId }
  })
  return c.json({ data: note })
})

// 删除笔记
app.delete('/:id', async (c) => {
  const userId = await getUserId(c)
  const id = c.req.param('id')

  const note = await prisma.note.findUnique({ where: { id } })
  if (!note || note.userId !== userId) {
    return c.json({ error: '不存在' }, 404)
  }

  await prisma.note.delete({ where: { id } })
  return c.json({ success: true })
})

export default app
```

2. 打开 `server/api/index.ts`，添加路由：

```typescript
import notesRoute from './routes/notes'

const routes = app.route('/', tasksRoute).route('/', notesRoute)
```

**完成！** 后端 API 已创建。

## 步骤 3：写 Hook

**问题**：前端如何调用 API？

**答案**：写 Hook，把 API 调用包装成简单函数。

**操作**：

创建文件 `hooks/use-notes.ts`：

```typescript
import { client } from '@/lib/api-client'
import { InferRequestType, InferResponseType } from 'hono/client'
import { useMutation, useQuery } from '@tanstack/react-query'
import { toast } from 'sonner'

// 定义类型（自动从 API 推导）
const $get = client.api.notes.$get
type GetResponseType = InferResponseType<typeof $get>

const $post = client.api.notes.$post
type PostRequestType = InferRequestType<typeof $post>['json']
type PostResponseType = InferResponseType<typeof $post>

const $delete = client.api.notes[':id'].$delete
type DeleteResponseType = InferResponseType<typeof $delete>

// 获取笔记列表
export function useNotes() {
  return useQuery<GetResponseType, Error>({
    queryKey: ['notes'],
    queryFn: async () => {
      const res = await $get()
      return res.json()
    },
  })
}

// 创建笔记
export function useCreateNote() {
  return useMutation<PostResponseType, Error, PostRequestType>({
    mutationFn: async (json) => {
      const res = await $post({ json })
      return res.json()
    },
    onSuccess: () => toast.success('创建成功'),
    onError: (e) => toast.error('失败：' + e.message),
  })
}

// 删除笔记
export function useDeleteNote() {
  return useMutation<DeleteResponseType, Error, string>({
    mutationFn: async (id) => {
      const res = await $delete({ param: { id } })
      return res.json()
    },
    onSuccess: () => toast.success('删除成功'),
    onError: (e) => toast.error('失败：' + e.message),
  })
}
```

**完成！** Hook 已创建，前端可以直接调用。

## 步骤 4：写页面

**问题**：用户看到什么界面？

**答案**：写页面组件，使用 Hook 获取数据和操作。

**操作**：

创建文件 `app/dashboard/notes/page.tsx`：

```typescript
'use client'

import { useState } from 'react'
import { useNotes, useCreateNote, useDeleteNote } from '@/hooks/use-notes'
import { Button } from '@/components/ui/button'
import { Input } from '@/components/ui/input'
import { Textarea } from '@/components/ui/textarea'
import { Trash2 } from 'lucide-react'

export default function NotesPage() {
  const [title, setTitle] = useState('')
  const [content, setContent] = useState('')

  const { data, isLoading } = useNotes()
  const createNote = useCreateNote()
  const deleteNote = useDeleteNote()

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault()
    if (!title.trim()) return
    createNote.mutate({ title, content })
    setTitle('')
    setContent('')
  }

  if (isLoading) return <div className="p-20 text-center">加载中...</div>

  const notes = data?.data || []

  return (
    <div className="min-h-screen py-20 px-6">
      <div className="max-w-3xl mx-auto">
        <h1 className="text-4xl font-bold mb-8">我的笔记</h1>

        {/* 创建笔记 */}
        <form onSubmit={handleSubmit} className="bg-white p-6 rounded-xl shadow-sm mb-8 space-y-4">
          <Input
            type="text"
            placeholder="标题"
            value={title}
            onChange={(e) => setTitle(e.target.value)}
            disabled={createNote.isPending}
          />
          <Textarea
            placeholder="内容"
            value={content}
            onChange={(e) => setContent(e.target.value)}
            disabled={createNote.isPending}
            className="h-32"
          />
          <Button
            type="submit"
            disabled={createNote.isPending || !title.trim()}
            className="w-full"
          >
            {createNote.isPending ? '创建中...' : '创建笔记'}
          </Button>
        </form>

        {/* 笔记列表 */}
        <div className="space-y-4">
          {notes.length === 0 ? (
            <div className="text-center py-12 text-gray-400">
              还没有笔记，创建第一个吧！
            </div>
          ) : (
            notes.map((note) => (
              <div key={note.id} className="bg-white p-6 rounded-xl shadow-sm">
                <div className="flex justify-between items-start">
                  <div>
                    <h3 className="text-lg font-semibold">{note.title}</h3>
                    {note.content && <p className="text-gray-600">{note.content}</p>}
                  </div>
                  <Button
                    variant="ghost"
                    size="icon"
                    onClick={() => deleteNote.mutate(note.id)}
                    disabled={deleteNote.isPending}
                  >
                    <Trash2 className="h-4 w-4 text-red-500" />
                  </Button>
                </div>
              </div>
            ))
          )}
        </div>
      </div>
    </div>
  )
}
```

**完成！** 页面已创建。

## ❓ 常见问题

**Q: 数据库迁移失败？**
```bash
rm dev.db
pnpm prisma migrate dev
```

**Q: 页面报错？**
- 检查文件路径
- 重启服务器：`Ctrl+C` → `pnpm dev`

**Q: API 返回 401？**
- 确保已登录
- 检查 `lib/api-client.ts` 是否有 `credentials: 'include'`

**Q: 页面显示 404？**
- 检查文件是否在 `app/dashboard/` 文件夹下
- 检查文件名是否是 `page.tsx`

**Q: 导航栏图标不显示？**
- 检查是否从 `lucide-react` 导入了图标
- 访问 https://lucide.dev/icons/ 查找图标名称
